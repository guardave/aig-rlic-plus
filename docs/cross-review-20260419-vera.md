## 4. Boundaries where Vera has been bitten

1. **Wave-3 NBER alpha 0.12 imperceptible (SL-2, April 2026).** VIZ-V2 prescribed 0.10–0.15 grey; Vera followed the rule exactly; rendering was invisible against the Streamlit background and dark data trace. **Root cause:** numeric prescription without perceptual validation. Rule was followed; rule was wrong. Addressed by V2 rev + META-PV (standards.md). **Residual gap:** perceptual PNG is produced but acceptance sign-off is not recorded anywhere machine-readable — Lesandro's eye is the only gate.
2. **Wave-3 Dot-Com canonical zoom → GATE-25 placeholder on Cloud (Bug B).** Chart file existed at the canonical path, JSON was parseable, but Ace's `load_plotly_chart` loader returned None. Vera's VIZ-V5 smoke test did not exist at the time; Ace's APP-ST1 did not exist. The *file* passed existence checks; the *loader* failed. **Root cause:** Vera tested "file loads" with her own Plotly import; Ace's loader has a different import path + caching layer + Streamlit mock. Addressed by VIZ-V5 + APP-ST1 + GATE-27, but they are **parallel** smoke tests — still no *joint* test that uses Ace's loader against Vera's output in one run.
3. **Wave-1.5 Granger chart fell back to Local Projections (S18-11).** Vera never owned a standalone Granger producer (VIZ-V3 did not yet exist); Evan never produced `granger_by_lag.csv` (ECON-E1 did not yet exist); Ace's charts.py had a legacy fallback that rendered whatever was nearest. Three-agent failure. **Root cause:** no per-method chart ownership contract. Addressed by VIZ-V3 + ECON-E1 + ECON-H4 + GATE-25. **Residual gap:** chart-type → method binding lives in three SOPs (Evan's H4 table, Vera's A3 table, Ace's render_method_block helper). Not a single source of truth.
4. **SL-3 chart-text drift (heatmap signals).** Vera updated heatmap signal list; Ray's "What the chart shows" paragraph was not updated. GATE-24 was introduced but is still a manual grep-check at handoff. **Residual gap:** Vera's regression_note entry and Ray's narrative-change entry live in two different files (`regression_note.md` vs `portal_narrative_*.md`). The Chart-Text Coherence rule says they must be coupled *in the same commit*, but there is no pre-commit hook enforcing it.
5. **Naming drift (pair-id-prefixed filenames).** Vera produced `hy_ig_v2_spy_correlation_heatmap.json`; Ace's loader expected `correlation_heatmap.json`. charts.py had a fallback; bug was latent. Hard-rejected in VIZ-A3 and APP-EP4. **Residual gap:** no CI test scans `output/charts/{pair_id}/plotly/` for pair-id-prefixed filenames and fails the build.
6. **Override silently replaced canonical baseline (potential, not observed).** META-ZI says overrides must *add* to canonical, never *replace*. No automated diff compares override against canonical.
7. **`_meta.json` caption vs Ray's narrative caption divergence.** Rule A5 says Vera flags but does not rewrite. In practice, QA time is tight; divergences ship. No gate item forces reconciliation at handoff.

## 5. Proposed new rules

> **Proposed VIZ-V6 — Joint Loader Smoke Test Contract**
> - Rule text: Before handoff, Vera MUST run the smoke test using the **same loader code path Ace uses in production** (i.e., import `app.components.charts.load_plotly_chart` under a Streamlit mock). A separate `plotly.io.read_json` check (current VIZ-V5) is necessary but not sufficient — the observed Wave-3 Dot-Com bug loaded cleanly via `read_json` and failed via Ace's loader. Vera's VIZ-V5 smoke test and Ace's APP-ST1 smoke test must share the underlying loader implementation; a PR that diverges them fails GATE-27.
> - Closes gap: Wave-3 Bug B (Dot-Com canonical zoom silent-None on Cloud)
> - Blocking?: Yes
> - Cross-reference: APP-ST1, GATE-27

> **Proposed VIZ-V7 — Coupled Chart+Narrative Regression Entry**
> - Rule text: When a chart update changes semantics (signal list, axis, transformation, color encoding), Vera's `regression_note.md` entry MUST include a `narrative_sync_ref` field citing the specific section/paragraph in `portal_narrative_{pair_id}.md` that was updated in the same commit. If no narrative update was needed, Vera writes `narrative_sync_ref: "none — verified with Ray at {timestamp}"` and Ray counter-signs by adding the same reference in her handoff acknowledgment. An entry without `narrative_sync_ref` fails GATE-24.
> - Closes gap: SL-3 chart-text drift (heatmap signals); GATE-24 partial fix
> - Blocking?: Yes
> - Cross-reference: RES-5 (regression prevention on reruns), GATE-24

> **Proposed VIZ-V8 — Canonical Chart-Type Registry as Machine-Readable Artifact**
> - Rule text: The canonical chart-type list (VIZ-A3 table) must be mirrored at `docs/chart_type_registry.json` with fields `{chart_type, required_columns, producer_rule_id, consumer_artifacts[], mandatory_diagnostics[]}`. Vera owns the JSON; Evan's ECON-H4 table and Ace's render_method_block helper must source chart_type keys from this file rather than hard-coding. Adding a new chart_type requires updating the JSON + a sop-changelog entry; ad-hoc new chart_types fail pre-commit.
> - Closes gap: S18-11 Granger fallback (three SOPs owned partial truth of the chart-type list); naming drift; future additions
> - Blocking?: Yes (blocking on new chart additions, advisory on existing)
> - Cross-reference: VIZ-A3, ECON-H4, APP-EP4

> **Proposed VIZ-V9 — Perceptual Sign-Off Log Entry**
> - Rule text: Every chart produced under META-PV (NBER shading, regime shading, alpha-dependent encoding) must append one line to `output/charts/{pair_id}/plotly/_perceptual_signoff.log` of the form `{timestamp} | {chart_name} | {encoding_type} | signed_off_by: {agent_name} | notes: {...}`. The signoff line is the acceptance artifact; a `_perceptual_check_*.png` without a corresponding signoff line fails GATE-27. For canonical charts (e.g., `output/_comparison/history_zoom_*`), log lives at `output/_comparison/_perceptual_signoff.log`.
> - Closes gap: SL-2 (Vera produced PNG but no record it was reviewed); provides gate reviewer something to audit
> - Blocking?: Yes for any chart touching color/alpha encoding
> - Cross-reference: META-PV, VIZ-V2, GATE-27

> **Proposed VIZ-V10 — Override-Diff Manifest for history_zoom**
> - Rule text: Whenever Vera ships a pair-override history_zoom chart (`output/charts/{pair_id}/history_zoom_{slug}.json`), she must also produce `output/charts/{pair_id}/history_zoom_{slug}_override_manifest.json` containing `{canonical_baseline_path, baseline_event_markers[], baseline_shading_spec, added_overlay[], retained_baseline[]}`. The manifest proves the override *adds* to canonical without silently replacing it, per META-ZI. A missing manifest on any override file is a GATE-26 failure.
> - Closes gap: META-ZI "override must start from canonical" is unverifiable today
> - Blocking?: Yes
> - Cross-reference: META-ZI, META-VNC, META-RNF
